# Rosetta app 设计澄清

当前Rosetta经历过多次迭代，已经基本符合我的要求。
但是由于迭代过程未能做好设计，匆忙实现，而导致代码可能混乱和不直观。

本文档的目的是，完整详细的阐述，Rosetta需要拥有的用户体验。

为将来重构本仓库，和清理与本文档观念不符的设计作为基准


## Rosetta 的核心理念

Rosetta 是 RWKV LLM 公司的产品，它对公司的价值是：

* 一款以 RWKV 为核心，最大化发挥 RWKV 模型特性，面向普通用户的产品 *

Rosetta 应该是一个完全本地，不依赖任何云端服务的桌面端翻译 app。

在用户安装完必要组件（翻译模型，pdf模组，等等）后，应该不再需要依赖网络（app 更新除外）



## 操作系统支持

Rosetta 应该支持不同的桌面端系统，他们是：

1. MacOS (苹果芯片)
2. Windows （英伟达显卡）
3. Windows （AMD 显卡）
4. Linux （英伟达显卡）

不需要支持的桌面端系统：

1. MacOs（intel 芯片）
2. Linux （AMD 显卡）

## 用户体验流程

### 1. 下载

用户从 Rosetta 官网下载对应操作系统的安装包

它的体积应该较小，不应该包含模型，pdf组件等重型资产


### 2. Onboard
用户在打开下载的安装包后，进入 Onboard 界面

它应该是一个简洁，直观的引导用户安装必要文件的向导 UI

安装必须是完全自动化的，用户只需要点击【安装】按钮。

所有需要安装的模块，需要考虑中国大陆用户无法文档连接国际互联网的网络环境。需要提供手动文件导入的兜底

除非影响用户进入 APP，所有在 Onboard 引导用户安装的文件，需要允许用户暂时跳过安装，并且将来在 app 的设置页面中安装。但是注意，暂时跳过安装应该是一个不被推荐的选项，在 UI 上应该弱化显示。
对于普通用户，完整完成 Onboard 的安装指引，可以为它避免将来大量的麻烦。

Onboard 需要引导用户安装的文件：

1. RWKV 翻译模型
2. PDF 组件

#### RWKV 翻译模型

Rosetta 应该可以根据用户操作系统，自动选择应该下载的模型

#### PDF 组件

Rosetta 应该可以根据用户操作系统，自动选择应该下载的 pdf组件



在用户安装完这两个模块后，应该检查这些模块的健康状态，确定用户进入 app 后，可以立即开始翻译任务。

立即可以开始翻译任务的定义：

1. rwkv 模型已经加载好，并且可以立即开始翻译生成
2. pdf 模组已经预热好，并且可以立即开始工作


在一切准备就绪后，onboard界面应该有一个清晰简洁的界面告诉用户一切就绪，并且提供一个显著的按钮退出onboard，进入主 app 界面

### 3. 主界面

当前主界面的 UI 是大致令人满意的，但是依旧有大量细节需要打磨

#### 主界面大致布局

##### 左侧侧边栏

左侧侧边栏分为三个区域：
1. 顶部导入新文件按钮

    点击后右侧主界面显示导入文件指引

2. 已经导入的文件列表（占据主要空间）

    按照导入时间排序，不应该在任何情况下在动态调整这个排序（用户很大程度是通过文件的位置来定位）
    
    需要有用icon区分 markdown，txt，pdf 文件

    需要一个简洁的ui呈现这个文件的翻译状态（未被翻译，翻译中，已翻译部分，完全翻译）

    当前被选中的文件需要有清晰的高亮

    用户可以通过右键文件，呼出菜单栏，进行重命名和删除

3. 底部前往设置页面的按钮


侧边栏可以被完全折叠。侧边栏需要支持系统原生半透明磨砂玻璃效果。


##### 右侧主界面
右侧主界面的总体布局是：

###### 上方操作栏
原文语言，目标译文语言选择。

翻译按钮

翻译状态，进度
