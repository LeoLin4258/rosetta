# Rosetta 0.1.0-beta.18

Rosetta beta.18 focuses on PDF translation correctness for the Windows
llama.cpp runtime, especially avoiding silently truncated local model output.

## Highlights

- Windows local PDF translation now launches the managed llama.cpp runtime with
  a larger default context, giving concurrent PDF translation requests more room
  to complete.
- llama.cpp responses that report truncation, hit the generation limit, or
  return empty text are now treated as translation failures instead of being
  accepted as complete output.
- PDF translation uses a llama.cpp-specific chunk profile and smaller retry
  backstops, so long PDF blocks are split more safely before they reach the
  local model.
- Rosetta now rejects a PDF translation invocation if unrecovered local model
  failures were reported, clears partial page artifacts from that attempt, and
  marks the affected pages as failed instead of ready.
- Very short numbered reference fragments are preserved directly, avoiding a
  known small-input repetition failure in the local model.
- First-install troubleshooting is easier: install-step diagnostics can be
  copied, and download source ranking now prefers mainland-accessible mirrors
  with simpler fallback behavior.

## Supported platforms

- Windows 10/11 x64. The recommended managed runtime uses llama.cpp Vulkan on
  supported AMD, Intel, and NVIDIA GPUs. Machines without a compatible Vulkan
  path may use CPU fallback or an explicitly configured external API.
- macOS 13 or newer on Apple Silicon remains supported through the existing
  local macOS runtime.

Unsigned Windows Preview installers may show a SmartScreen or
unknown-publisher warning on first install unless an Authenticode certificate
is configured during release.

Rosetta remains local-first. Documents, translations, job caches, prompts, and
runtime logs are not uploaded by the release or update system.
