# Rosetta 0.1.0-beta.15

Rosetta beta.15 focuses on the Windows local runtime. The Windows Preview
release now defaults to a llama.cpp Vulkan runtime so more Windows x64
machines can run local translation without requiring an NVIDIA CUDA GPU.

## Highlights

- Windows local translation now defaults to llama.cpp Vulkan with the RWKV v7
  G1d 0.4B Translate GGUF model.
- Windows hardware coverage expands beyond NVIDIA SM75+ to supported AMD,
  Intel, and NVIDIA Vulkan devices.
- RWKV Lightning CUDA remains available as a secondary NVIDIA runtime, but is
  no longer the recommended Windows default.
- First-run installation automatically probes download sources and silently
  chooses a fast available source.
- Runtime installation validates Vulkan availability with `llama-server.exe
  --list-devices`.
- AMD Radeon 780M and similar devices that lack required Vulkan extensions can
  fall back to CPU mode without crashing the Vulkan backend.
- Document and PDF translation now use the RWKV translate model's native prompt
  format through llama.cpp `/completion`, improving translation reliability.
- Windows translation parallelism is raised for better throughput on the small
  0.4B local model.

## Supported platforms

- Windows 10/11 x64. The recommended managed runtime uses llama.cpp Vulkan on
  supported AMD, Intel, and NVIDIA GPUs. Machines without a compatible Vulkan
  path may use CPU fallback or an explicitly configured external API.
- macOS 13 or newer on Apple Silicon remains supported through the existing
  local macOS runtime.

This Windows Preview installer is not Authenticode-signed unless a signing
certificate is configured during release. Unsigned installers may show a
SmartScreen or unknown-publisher warning on first install.

Rosetta remains local-first. Documents, translations, job caches, prompts, and
runtime logs are not uploaded by the release or update system.
